# VK09 Maintenance Page

Стильная страница «Режим обслуживания» для сайта-портфолио — с интерактивным терминалом, прогресс-баром разработки и скрытыми пасхалками.

## Описание

Страница-заглушка, которую можно показывать пользователям, пока основной сайт находится в разработке или на обслуживании. Вместо скучного «Сайт временно недоступен» — полноценный интерфейс с системным монитором, таймлайном этапов и настоящим терминалом.

### Возможности

- **Интерактивный терминал** — настоящая командная строка с набором команд (`help`, `status`, `version`, `stack`, `uptime`, `clear`)
- **Прогресс-бар разработки** — автоматически рассчитывается из таймлайна этапов, с обратным отсчётом и счётчиком данных
- **Системный монитор** — правая панель с информацией о сборке, таймлайном шагов и потоком лог-сообщений
- **Таймлайн этапов** — визуализация этапов разработки (дизайн, вёрстка, контент, тестирование, деплой)
- **Пасхалки** — 3 скрытых секрета для любопытных посетителей
- **Адаптивный дизайн** — корректное отображение на мобильных и десктопных устройствах
- **Конфигурация через файл** — все тексты, контакты и этапы настраиваются в одном конфиге

## Скриншот

Общий вид страницы: тёмный интерфейс с панелью контента слева (заголовок, описание, прогресс-бар, кнопки контактов) и системным монитором справа (версия сборки, таймлайн, терминал).

## Быстрый старт

### Требования

- **Node.js** 18+ (рекомендуется 20+)
- **npm** 9+ (или любой другой пакетный менеджер)

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/vkambulov/vk09-maintenance.git
cd vk09-maintenance

# Установить зависимости
npm install

# Запустить dev-сервер
npm run dev
```

Откройте [http://localhost:5173](http://localhost:5173) в браузере.

### Сборка для продакшена

```bash
npm run build
```

Статические файлы будут в папке `dist/`. Их можно разместить на любом хостинге (Vercel, Netlify, GitHub Pages, Nginx и т.д.).

### Предпросмотр сборки

```bash
npm run preview
```

## Конфигурация

Все настройки страницы находятся в одном файле:

**`src/config/maintenance.config.ts`**

| Секция | Что настраивается |
|---|---|
| `progressMeta` | Оставшееся время, начальный объём данных, скорость |
| `release` | Версия сборки, Node.js, фреймворка, контент-модуля |
| `stack` | Технологический стек (отображается в команде `stack`) |
| `timelineSteps` | Этапы разработки — название, статус (`completed` / `active` / `pending`), детали |
| `contacts` | Email и Telegram — ссылки и подписи кнопок |
| `links` | Внешние ссылки (GitHub) |
| `maintenance` | Заголовок страницы, описание, текст футера |

### Пример: обновить этапы разработки

```typescript
export const timelineSteps: TimelineStep[] = [
  {
    title: 'Дизайн и прототип',
    subtitle: 'Завершено',
    status: 'completed',
  },
  {
    title: 'Верстка и компоненты',
    subtitle: 'В процессе',
    status: 'active',
    detail: 'Сборка UI-компонентов...',
    detailSecondary: 'Компоненты: 5/12',
  },
  // ...
]
```

Процент прогресс-бара пересчитается автоматически.

## Терминал

В правой нижней части страницы расположен интерактивный терминал. Введите `help`, чтобы увидеть список доступных команд:

| Команда | Описание |
|---|---|
| `help` | Список доступных команд |
| `status` | Текущий статус системы и прогресс сборки |
| `version` | Версия сборки, Node.js, фреймворка |
| `stack` | Технологический стек проекта |
| `uptime` | Время работы с начала обслуживания |
| `clear` | Очистить вывод терминала |

Терминал также автоматически выводит рандомизированные лог-сообщения сборки каждые 5 секунд, создавая эффект «живого» процесса.

## Использование в своём проекте

1. Форкните или клонируйте репозиторий
2. Отредактируйте `src/config/maintenance.config.ts` — замените контакты, тексты и этапы на свои
3. При необходимости измените цветовую схему в компонентах (основной акцент — `#ff8c00`)
4. Соберите проект (`npm run build`) и разместите `dist/` на хостинге
